# Ribbon Sync 未登录提示与登录引导设计说明

日期：2026-04-22

状态：设计已确认，进入实施

## 1. 目标

调整 Ribbon Sync 在业务系统未登录场景下的行为，使其：

- 将 `401 Unauthorized` 和 `403 Forbidden` 都视为“当前未登录”
- 对外统一提示文案为 `当前未登录，请先登录`
- 在所有 Ribbon Sync 入口提供一个可直接发起登录的 `点我登录` 按钮
- 继续复用现有 Ribbon 登录流程，不新增第二套 SSO 登录实现

## 2. 适用范围

本次变更覆盖以下 Ribbon Sync 入口：

- 项目下拉框加载项目列表
- 初始化当前表
- 部分下载
- 部分上传
- 现有通过 `RibbonSyncController` 进入、并可能落到业务系统接口的其他同步动作

本次不覆盖：

- task pane 设置页里的登录态展示规则
- Agent / Skill 相关 HTTP 调用的认证错误交互
- 自动重放失败前的同步动作

## 3. 当前问题

当前行为存在三个问题：

1. 只有 `401` 被识别为“未登录”，`403` 仍会落为普通失败
2. 认证失败文案对用户暴露为 `业务系统未登录，请先登录后重试。`，不符合新的交互要求
3. 项目列表失败和同步动作失败都只会弹普通消息框，用户需要自己再去找 Ribbon 上的 `登录` 按钮

## 4. 方案

本次采用“认证异常类型化 + 统一登录提示对话框 + 复用现有 Ribbon 登录入口”的方案。

### 4.1 认证失败识别

`CurrentBusinessSystemConnector` 在收到以下状态码时，统一抛出专用认证异常：

- `401`
- `403`

该异常面向 UI 层暴露的消息固定为：

- `当前未登录，请先登录`

### 4.2 统一登录提示对话框

新增一个小型 WinForms 对话框，职责只有：

- 显示认证失败文案
- 提供 `点我登录` 按钮
- 提供关闭入口

对话框不负责自己实现登录，只负责返回用户是否选择去登录。

### 4.3 Ribbon 登录复用

新增一个可复用的 Ribbon 登录执行入口，供两个地方共用：

- 现有 Ribbon `登录` 按钮点击
- 统一登录提示对话框点击 `点我登录`

登录成功后：

- 项目下拉框场景：立即重新加载项目列表并刷新下拉框文本
- 同步动作场景：只完成登录，不自动重试刚才失败的操作；用户手动重新触发

这个选择是为了避免在失败上下文里隐式重放下载/上传/初始化操作。

## 5. UI 规则

### 5.1 项目下拉框加载项目列表

当项目接口返回 `401/403` 时：

- 项目下拉框状态仍显示 `请先登录`
- 弹出统一登录提示对话框
- 如果用户点击 `点我登录` 并登录成功，立即重新加载项目列表
- 如果用户取消或关闭登录，不再继续重试

### 5.2 初始化 / 下载 / 上传

当上述动作内部命中认证异常时：

- 不走普通错误对话框
- 直接弹出统一登录提示对话框
- 如果用户点击 `点我登录` 并登录成功，本次动作结束，由用户重新触发
- 如果用户取消或关闭登录，本次动作结束

## 6. 测试要求

需要补充并验证以下行为：

- `401` 返回时仍被识别为认证失败
- `403` 返回时也被识别为认证失败
- 认证异常文案固定为 `当前未登录，请先登录`
- Ribbon 配置测试覆盖统一登录提示入口与登录复用入口
- RibbonSyncController 测试覆盖认证异常不再落到普通 `ShowError`
